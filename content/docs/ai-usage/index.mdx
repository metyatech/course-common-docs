# 生成AIの適正な利用方法について

## 生成AIとは

**生成AI** とは、質問や指示を普通の言葉で入力すると、文章・コード・画像などを自動で作ってくれるAIのことです。

代表的なサービスには次のようなものがあります。

- **ChatGPT**（OpenAI） — 文章やコードの生成・質問への回答など、幅広い用途に使える対話型AI
- **Gemini**（Google） — Google が提供する対話型AI。Google 検索との連携が特徴
- **Claude**（Anthropic） — 長い文章の読解や丁寧な説明が得意な対話型AI
- **GitHub Copilot**（GitHub） — プログラミング向けのAI。エディタ上でコードを提案してくれるほか、チャットで質問もできる

いずれも、**質問や指示を出すと、対話的にAIが文章（質問への回答、コードなど）や画像などを生成してくれる** というサービスです。

さらに最近では、**AIエージェント**（Claude Code、Codex、GitHub Copilot の Agent モードなど）と呼ばれるツールも登場しています。こちらはチャットでの会話だけでなく、**ファイルの編集やコマンドの実行までAIが自動で行う** のが特徴です。便利な反面、一度に大量の変更が行われるため、**自分で内容を把握する意識がより大切** になります。

どちらを使う場合でも、このページのルールは同じように当てはまります。

このページでは、このような **生成AIの、授業や試験での適正な利用方法** を説明します。

---

:::important[先に結論]
- **生成AI** は原則使ってOKです。
- 本質的には、**コードが何をしていて、どこを直せばよいか** を理解していることが大切です。
- **自分で扱えないものを、そのまま提出するのはNG** です。
:::

大事なのは、**あなた自身が何らかの概念を知ったり、理解したりして成長すること** です。  
この授業では、**授業中も試験中も**、生成AIを学習や作業の道具としてどんどん活用してください。

## 良い例

**良い例**は、*理解する*・*下書きを作る*・*比較して学ぶ* です。

### 分からないことを理解するために使う

:::tip[良い例]
- **エラーを理解したい**: 「このエラー文の意味を、やさしい言葉で説明して」
- **コードの目的を知りたい**: 「このコードは何を実現していて、どこが重要か説明して」
- **説明を分かりやすくしたい**: 「この文章を、高校生にも分かる言い方に直して」
:::

:::note[なぜ？]
- **理由**: 最終的なゴールが **自分が理解すること** にあるからです。
- **たとえば**: エラーの意味をAIに聞いて、理解したうえで修正する。これならちゃんと意味を理解しています。
:::

### 下書きを作ってもらい、自分で直す

:::tip[良い例]
- **見出しの案がほしい**: 「レポートの見出し案を3つ出して」
- **たたき台がほしい**: 「このコードのたたき台を作って」
- **別案を見たい**: 「別の書き方の例を2つ見せて」
:::

:::note[なぜ？]
- **理由**: 下書きであれば、そこから自身で手を加えることになり、その過程で自身の **理解** が必要になるからです。
- **たとえば**: AIが作ったコードのたたき台には、<u>想定外の動き</u> や不要な処理が入ることがあります。自分で読み直して、**何をするコードか** と **どこを変えるべきか** を説明できるところまでになれば、全体の内容は把握できていることになります。
:::

### 比較して学ぶ

:::tip[良い例]
- **模範解答と比較したい**: 「この書き方と模範解答の違いを教えて」
- **改善点を知りたい**: 「このコードの改善方法を3パターン挙げて」
:::

:::note[なぜ？]
- **理由**: 比較した結果を自身で解釈することで、**自身の学びにつながる**からです。
- **たとえば**: 自身の回答と模範解答を見比べることで、自身のコードの不備や改善点に気付くことができます。こうした気づきは次に活かすことができます。
:::

## 悪い例

**悪い例**は、*確認しない*・*説明できない*・*情報漏えいする* です。

### AIの答えを確認せずに信じる

:::caution[悪い例]
- **未実行で使う**: <u>実行していないコード</u> を、そのまま正しいと思って使う。
- **無検証で信じる**: AIが書いた説明を、他の手段で得られる（教科書、授業資料、Webなどからの）情報と照らし合わせずにうのみにする。
- **文章をそのまま使う**: AIが書いた文章の内容を、<u>自分の言葉で言い直さない</u> まま提出する。
:::

:::note[なぜ？]
- **理由**: 生成AIは、*もっともらしい間違い（ハルシネーション）* を出したり、あなたの意図と異なるものを出すこともあるからです。
- **たとえば**: **存在しない関数名を使ったり、関数の引数や動作を間違えて説明したり**、作って欲しかった文章と全然違う文章を作成したりします。見た目が自然なので気づきにくいですが、実際に実行してみたり、他の情報源と照らし合わせて確認することは非常に重要です。
:::

### 動作を説明できないまま提出する

:::caution[悪い例]
- **そのまま提出**: AIが作ったコードを、<u>何をするものか分からないまま</u> 提出する。
:::

:::note[なぜ？]
- **理由**: 提出したものは、**自分の成果物** として扱われるため、何か問題があればそれはあなたの責任となるからです。
- **補足**: 全行を手書きできる必要はありません。ただし、**自分が出すものが何をしているか** くらいは分かるようにしておきましょう。
:::

### 入力してはいけない情報を入れる

:::caution[悪い例]
- **ログイン情報**: 学校アカウントの情報やパスワードを入力する。
- **個人情報**: 自分や他人の成績、住所、電話番号などの個人情報を入力する。
:::

:::note[なぜ？]
- **理由**: 入力した内容は **AIサービスのサーバーに送られる** ため、自分だけの情報ではなくなる可能性があります。
- **たとえば**: 「ログインできないから見て」と言って、<u>IDやパスワードをそのまま入力する</u> のは危険です。困ったときは、**情報を伏せた状態** で質問してください。
:::

## 試験中の使い方

**試験中** も考え方は同じです。AIの答えをそのまま写しても、自分の力にはなりません。

:::note[基本ルール]
中間試験・定期試験でも、検索と生成AIの利用はOKですが、適正な利用を心がけましょう。
:::

:::tip[良い例]
- **候補を出してもらう**: 「このコードのエラーの原因を3つ挙げて」
- **見落としを確認する**: 「この説明で見落としている点がないか確認して」
- **修正方針を整理する**: 「このコードが何をしていて、どこを直せばよいか整理して」
:::

:::caution[悪い例]
- **そのまま写す**: AIが出した答えを、<u>自分で確認せずに</u> そのまま答案に書く。
- **説明できないまま提出する**: AIが作ったコードや文章を、<u>何をするものか分からないまま</u> 提出する。
:::

:::note[なぜ？]
- **理由**: 大事なのは、**自分が理解・判断できているか** です。
- **たとえば**: AIにエラーの原因を出してもらい、自分で絞って直す — これは丸投げではなく、自分の判断です。一方、AIの答えをそのまま写すだけでは、自分が理解していることを示せません。
:::

## まとめ

次の3点を意識して、生成AIをうまく活用していきましょう。

:::important[チェックポイント]
1. 提出するコードや文章の **内容を理解していますか？**
2. AIの出力を **確認しましたか？**
3. **入れてはいけない情報** を入力していませんか？
:::

この3つにすべて「はい」と言えるなら、**安全な使い方** です。AIをうまく使って、自分の力を伸ばしていきましょう。
