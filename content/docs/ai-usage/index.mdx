# 生成AIの利用

「生成AIは使っていいの？」と迷ったら、まずは次の3つだけ覚えてください。

:::important[先に結論]
- 生成AIは原則使ってOKです。
- 手書きできることより、何をしていて、どこを直せばよいかを自分で説明できることが大切です。
- 自分で扱えないものを、そのまま提出するのはNGです。
:::

この授業では、授業中も試験中も、生成AIを学習や作業の道具として使ってかまいません。

この授業で大事なのは、「コードを全部手書きできること」よりも、「AIを使っても、自分で動作・目的・直し方を判断できること」です。

担当教員から別の指示があった場合は、その指示を優先してください。

## 使ってよい使い方

### 分からないことを理解するために使う

:::tip[OK例]
- 「このエラー文の意味を、やさしい言葉で説明して」
- 「このコードは何を実現していて、どこが重要か説明して」
- 「この文章を、高校生にも分かる言い方に直して」
:::

:::note[なぜ？]
意味をつかむために使うなら、AIは「答えを丸ごと出す機械」ではなく「理解を助ける道具」になります。今の時代に大事なのは、全行を暗記することよりも、「このコードは何をするのか」「どこが重要か」をつかむことです。たとえば、AIに概要を説明してもらってから自分で直すなら、最後に考えているのは自分です。
:::

### 下書きを作ってもらい、自分で直す

:::tip[OK例]
- 「レポートの見出し案を3つ出して」
- 「このコードのたたき台を作って」
- 「別の書き方の例を2つ見せて」
:::

:::note[なぜ？]
下書きは出発点だからです。AIが作ったたたき台には、不要な処理や想定外の動きが入ることがあります。自分で読み直して、「何をするコードか」「どこを変えたか」を言えるところまでやって、はじめて自分の学びになります。
:::

### 比べて学ぶ

:::tip[OK例]
- 「この書き方と別解の違いを教えて」
- 「この説明のどこが分かりにくいか教えて」
- 「このコードの改善点を3つ挙げて」
:::

:::note[なぜ？]
比較すると、1つの答えをうのみにせずに考えられます。たとえば、自分のコードと別解を見比べると、「なぜこの書き方の方が読みやすいのか」が分かります。これは次の課題にも残る学びです。
:::

## 使ってはいけない使い方

### 動作や直し方を説明できないまま提出する

:::caution[NG例]
- AIが作ったコードを、自分で何をするものか確認せずにそのまま提出する。
- 「どこを直せばよいか」を自分で言えないまま提出する。
- AIが作った文章を、自分の言葉で説明できないまま提出する。
:::

:::note[なぜ？]
提出物（＝ 仕事の成果物）は「自分が責任をもつもの」である必要があるからです。ここでいう理解は、全行を手書きできることではありません。少なくとも、「何をするものか」「どこが重要か」「壊れたらどこを見て直すか」が言えないと、自分で扱える状態とは言えません。
:::

### AIの答えを確認せずに信じる

:::caution[NG例]
- 実行していないコードを、そのまま正しいと思って使う。
- AIが書いた説明を、教科書や授業内容と比べずにそのまますべて信じて頭に入れる。
:::

:::note[なぜ？]
生成AIは便利ですが、もっともらしい間違いも出します。たとえば、存在しない関数名を書いたり、授業で扱っていない仕様を混ぜたりすることがあります。見た目が自然でも、中身は普通に間違います。だから「どこを確認すれば安全か」を人が持っている必要があります。
:::

### 入力してはいけない情報を入れる

:::caution[NG例]
- 学校アカウントの情報やパスワードを入力する。
- 未公開の成績情報や個人情報を入力する。
:::

:::note[なぜ？]
一度入力した情報は、自分の手元だけで完結しない可能性があるからです。たとえば、「ログインできないから見て」と言って、IDやパスワードをそのまま入力するのは危険です。困ったときは、情報を伏せた状態で質問してください。
:::

## 試験中の使い方

:::note[基本ルール]
中間試験・定期試験でも、担当教員から別の指示がない限り、検索と生成AIの利用を認めます。
:::

:::tip[OK例]
- 「このコードのエラー候補を3つ挙げて」
- 「この説明で見落としている点がないか確認して」
- 「このコードが何をしていて、どこを直せばよいか整理して」
:::

:::caution[NG例]
- 「この問題の答えをそのまま全部作って」
- AIが出した答えを、自分で確認せずにそのまま答案に書く。
:::

:::note[なぜ？]
この授業で見たいのは、AIを使ったかどうかよりも、AIを使っても自分で判断できるかです。たとえば、AIにエラー候補を出してもらい、自分で原因を絞って直すなら、それは丸投げではなく判断です。反対に、AIの答えをそのまま写すだけでは、「自分で動作や修正方針を持てているか」が確認できません。
:::

## 迷ったときは、この3つで判断してください

:::important[チェックポイント]
1. この提出物が何をするものか、自分の言葉で説明できますか。
2. おかしいときに、どこを見てどう直すかを説明できますか。
3. 入力してはいけない情報を入れていませんか。
:::

この3つにすべて「はい」と言えるなら、かなり安全な使い方です。
