# 生成AIの利用

「生成AIは使っていいの？」と迷ったら、まずは次の3つだけ覚えてください。

:::important
- **生成AIは原則使ってOKです。**
- **意味が分からないまま、そのまま提出するのはNGです。**
- **最後は自分で説明できる状態にしてください。**
:::

この授業では、授業中も試験中も、生成AIを学習や作業の道具として使ってかまいません。

担当教員から別の指示があった場合は、その指示を優先してください。

## 使ってよい使い方

### 1. 分からないことを理解するために使う

たとえば、次のような使い方です。

- 「このエラー文の意味を、やさしい言葉で説明して」
- 「このコードが何をしているか、1行ずつ説明して」
- 「この文章を、高校生にも分かる言い方に直して」

**なぜOKなのか**

抽象: これは、自分の理解を深めるための使い方だからです。答えを受け取ることより、意味をつかむことが中心になっています。

具体: エラー文の意味をAIに聞いて、自分でコードを直すなら、最終的に考えているのは自分です。この使い方なら、理解も残ります。

### 2. 下書きを作ってもらい、自分で直す

たとえば、次のような使い方です。

- 「レポートの見出し案を3つ出して」
- 「このコードのたたき台を作って」
- 「別の書き方の例を2つ見せて」

この場合も、そのまま出すのではなく、自分で読み直して直してください。

**なぜそのまま出してはいけないのか**

抽象: 下書きは出発点であって、完成品ではないからです。自分で直す過程がないと、理解も判断も自分のものになりません。

具体: AIが作ったコードのたたき台に不要な処理や授業で扱っていない書き方が入ることがあります。自分で直さず提出すると、動いても説明できない状態になります。

### 3. 比べて学ぶ

たとえば、次のような使い方です。

- 「この書き方と別解の違いを教えて」
- 「この説明のどこが分かりにくいか教えて」
- 「このコードの改善点を3つ挙げて」

**なぜこの使い方がよいのか**

抽象: 比較すると、1つの答えをうのみにせずに考えられるからです。AIを「正解を出す機械」ではなく、「考える材料を増やす道具」として使えます。

具体: 自分のコードと別解を見比べると、「なぜこの書き方の方が読みやすいのか」が分かります。これはその場しのぎではなく、次の課題にも残る学びです。

## 使ってはいけない使い方

### 1. 意味が分からないまま提出する

これはNGです。

- AIが作ったコードを、自分で読まずにそのまま提出する。
- AIが作った文章を、内容を確認せずにそのまま提出する。

**なぜNGなのか**

抽象: 提出物は「自分が理解して扱えるもの」である必要があるからです。意味が分からないまま出すと、学習にも評価にもなりません。

具体: AIが作ったコードがたまたま動いても、「この行は何をしているのか」と聞かれて答えられないなら、その提出物は自分の力を示していません。

### 2. AIの答えを確認せずに信じる

これはNGです。

- 実行していないコードを、そのまま正しいと思って提出する。
- AIが書いた説明を、教科書や授業内容と比べずにそのまま使う。

**なぜ確認が必要なのか**

抽象: 生成AIは便利ですが、もっともらしい間違いも出します。だから最後の確認は人がやる必要があります。

具体: AIが存在しない関数名を書いたり、授業で習っていない仕様を混ぜたりすることがあります。見た目が自然でも、中身は普通に間違います。

### 3. 入力してはいけない情報を入れる

これはNGです。

- 学校アカウントの情報やパスワードを入力する。
- 未公開の成績情報や個人情報を入力する。

**なぜNGなのか**

抽象: 一度入力した情報は、自分の手元だけで完結しない可能性があるからです。学習のためでも、出してはいけない情報があります。

具体: 「ログインできないから見て」と言って、IDやパスワードをそのまま入力するのは危険です。困ったときは、情報を伏せた状態で質問してください。

## 試験中の使い方

:::info
**中間試験・定期試験でも、担当教員から別の指示がない限り、検索と生成AIの利用を認めます。**
:::

試験中も考え方は同じです。  
「使うこと」自体は問題ではなく、「自分で理解して扱えているか」が大事です。

**なぜこの考え方なのか**

抽象: この授業では、AIを使ったかどうかよりも、AIを使っても自分で判断できるかを大切にしているからです。

具体: AIにエラー候補を出してもらい、その中から自分で原因を特定して直すなら、評価されるのは「丸投げ」ではなく「自分の判断」です。

たとえば、次のような使い方はOKです。

- 「このコードのエラー候補を3つ挙げて」
- 「この説明で見落としている点がないか確認して」

逆に、次のような使い方はNGです。

- 「この問題の答えをそのまま全部作って」
- AIが出した答えを、自分で確認せずにそのまま答案に書く。

**なぜこれはNGなのか**

抽象: 試験で見たいのは、最終的に自分で内容を扱えるかどうかだからです。AIの出力をそのまま移すだけでは、その確認ができません。

具体: AIが作った答案を写しただけだと、その場では点が取れたように見えても、後で同じ内容を口頭で説明したり応用したりできません。それでは実力確認になりません。

## 迷ったときは、この3つで判断してください

1. この内容を、自分の言葉で説明できますか。
2. この答えが間違っていたとき、自分で直せますか。
3. 入力してはいけない情報を入れていませんか。

この3つにすべて「はい」と言えるなら、かなり安全な使い方です。
